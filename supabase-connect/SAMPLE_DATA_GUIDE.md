# Sample Data Insertion Guide

This directory contains scripts and SQL files to populate your Supabase database with 50 sample records for each table.

## Files Included

1. **insert_sample_data.sql** - Complete SQL script with all 200 sample records
2. **populate_sample_data.py** - Python script using psycopg2 (requires network access)
3. **api/management/commands/populate_sample_data.py** - Django management command

## Method 1: Using SQL Editor (Recommended - No Network Issues)

The easiest way to populate the database is using the Supabase SQL Editor:

### Steps:
1. Go to your Supabase Dashboard
2. Navigate to **SQL Editor**
3. Click **"New Query"**
4. Copy the entire contents of `insert_sample_data.sql`
5. Paste into the SQL Editor
6. Click **"Run"** button
7. Wait for completion (should take a few seconds)

**Result**: 50 records each in:
- api_meeting
- api_participant
- api_task
- api_document

---

## Method 2: Python Direct Insertion Script

If you gain network access to Supabase from your environment:

```bash
source venv/bin/activate
python populate_sample_data.py
```

This script:
- Uses psycopg2 to connect directly to Supabase
- Generates realistic data using Faker library
- Creates proper foreign key relationships
- Prints progress as it executes

---

## Method 3: Django Management Command

Once Django can connect to the database:

```bash
python manage.py populate_sample_data
```

This command:
- Uses Django ORM for inserts
- Properly handles relationships
- Provides formatted output with progress indicators

## Troubleshooting

### Issue: "relation does not exist" error
- **Solution**: Make sure the tables were created first using the table creation SQL

### Issue: Foreign key constraint violations
- **Solution**: Ensure api_meeting records exist before inserting participants/tasks/documents

### Issue: Connection refused (Python script)
- **Solution**: Network connectivity issue. Use Method 1 (SQL Editor) instead
